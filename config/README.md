# 配置目录说明

这个目录用于承接后续的配置重构。

当前项目仍然通过根目录下的 `config.py` 读取配置，因此这里暂时只放模板和说明，不参与实际运行。

这样做的目的：

- 先把新结构搭起来
- 避免立即引入 `config/__init__.py` 后与现有 `config.py` 发生导入冲突
- 等下一步正式切换配置加载逻辑时，再一次性处理导入兼容

计划中的文件：

- `settings.example.py`
  - 示例配置模板
  - 只放占位符，不放真实密钥

- `settings.local.py`
  - 本地真实配置
  - 后续创建后应被 `.gitignore` 忽略

注意：

- 在真正切换前，不要手动创建 `config/__init__.py`
- 否则现有代码中的 `from config import ...` 很可能会优先导入这个目录，导致项目运行异常
