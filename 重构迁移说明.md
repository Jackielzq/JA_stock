# 重构迁移说明

## 目的

这次重构的目标是把项目从“源码、配置、输入文件、运行产物混在一起”的状态，整理成更适合长期维护的结构。

重点解决的问题：

- 根目录结构过于平铺
- `config.py` 直接保存敏感信息
- `monitor_pool.xlsx` 路径硬编码
- `output/` 下所有报告平铺堆积
- 缓存和历史报告长期污染工作区

## 本次已完成的调整

### 1. 配置结构切换

旧结构：

- 根目录 `config.py`

新结构：

- `config/__init__.py`
- `config/settings.example.py`
- `config/settings.local.py`

说明：

- 现在项目统一通过 `from config import ...` 进入新配置入口
- 本地真实配置放在 `settings.local.py`
- 示例模板放在 `settings.example.py`
- 根目录旧 `config.py` 已删除

### 2. 监控池输入文件迁移

旧路径：

- `utils/monitor_pool.xlsx`

新结构：

- `data/input/monitor_pool.sample.xlsx`
- `data/input/monitor_pool.local.xlsx`

说明：

- 现在统一通过配置常量 `MONITOR_POOL_FILE` 读取
- 程序优先读取本地 `monitor_pool.local.xlsx`
- 如果本地文件不存在，则回退到 `monitor_pool.sample.xlsx`
- `main.py` 和 `core/monitor_manager.py` 都已经切换到这套逻辑

### 3. 输出目录拆分

旧结构：

- `output/` 下所有 HTML/PDF 平铺

新结构：

- `output/daily_review/`
- `output/daily_news/`
- `output/monitor/`
- `output/stock_selection/`
- `output/backtest/`
- `output/pdf/`
- `output/img/`

说明：

- `main.py`
- `report/renderer.py`
- `services/github_service.py`
- `services/deploy_hexo.py`

这些模块都已经改成按新目录读取和输出。

### 4. 缓存和历史报告清理

已处理：

- 删除 Python 缓存目录 `__pycache__`
- 清理一周以前的旧报告
- 保留近期报告与结构图文件

## GitHub 说明

当前项目中的 `github` 模式用途是：

- 上传静态复盘页面到展示仓库

它不是：

- 上传整个源码项目
- 管理源码版本历史

如果后续要把整个项目源码同步到 GitHub，建议单独作为源码仓库流程处理，不和当前静态页面发布逻辑混用。
